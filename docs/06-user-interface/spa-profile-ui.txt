Spa Profile UI Documentation
=============================

This document records the current interface layout, section placement, and behavior of the public Spa Profile page as implemented at /spa/{establishment_slug}. It documents interface direction only: visual layout, section order, and interaction behavior. It does not define data fields, database structure, permissions, or lifecycle rules; those remain in the Spa Profile Guide (docs/05-directory/spa-profile.txt). Where this document and the Spa Profile Guide differ, the Spa Profile Guide is the content authority and this document should be reconciled to it rather than treated as a silent override.

Status: this is a first implementation-based draft (planning version), not yet a thoroughly reviewed interface specification. It is written from the built page so the interface has a durable record, and remains open to revision.

Page Layout Overview
---------------------

The spa profile page is divided into three layout areas: a dark Identity Header spanning the full width, a sticky in-page Tab Navigation bar directly beneath it, and a two-column Body below that pairs the main content column with a sticky Booking panel in the right rail on wide viewports. The page reuses the site-wide header and footer from the homepage layout.

--------------------------------------------------------------------------------

1. Identity Header
===================

Placement: Top of the page, full width, dark Ink & Ember background (matches the homepage hero treatment) with a breadcrumb trail (Home › Spas › {spa name}) above the content.

Shows, when available: a square brand mark or fallback icon tile, Verified and Claimed Listing badges, the spa name, Spa Type and Spa Market Class and area on one line, a rating summary with review count, current open/closed state with closing time, and feature badges (Home Service Available, Gift Certificates, Resorts & Relaxation). A compact photo grid preview sits on the right on wide viewports with a "View all photos (N)" overlay on the last tile; a full gallery view is not yet a separate page or tab.

--------------------------------------------------------------------------------

2. Primary Actions
====================

Placement: Bottom of the Identity Header, full width.

A horizontal action row: Book Now (primary, links to the booking panel), Contact, Get Directions, Save, Follow, Share, and a right-aligned Suggest a Correction link. Report is not yet present as a distinct action and should be added when the moderation/report flow exists.

--------------------------------------------------------------------------------

3. Tab Navigation
===================

Placement: Immediately below the Identity Header, sticky beneath the site header while scrolling.

An in-page anchor tab bar: Overview, Services & Rates, Therapists, Facilities, Reviews, Location. Clicking a tab scrolls to the matching section rather than loading a new page or an ajax panel. This is a reduced set compared to the Spa Profile Guide's eleven-item Main Sections/Tabs list: Availability and Booking is represented as the always-visible Booking panel rather than its own tab, Gallery and Public Documents is represented only by the header photo-grid preview, and Promotions and Updates, and About/Verification/Sources are not yet separate tabs (Verification appears as an untabbed section at the end of the main column; Promotions and Updates is not yet implemented on this page). These gaps are deferred, not rejected.

--------------------------------------------------------------------------------

4. Overview
=============

Placement: Main column, first section.

Shows the about/description text and an amenities grid (icon, label, and an Available/Accepted status) sourced from the establishment's facility list.

--------------------------------------------------------------------------------

5. Services & Rates
======================

Placement: Main column, after Overview.

A list of provider menu items, each showing an icon tile, service name, duration in minutes, and price. A last-confirmed date appears below the list per the Spa Profile Guide's requirement that displayed prices carry a visible confirmation date.

--------------------------------------------------------------------------------

6. Therapists
===============

Placement: Main column, after Services & Rates.

A card grid of therapists affiliated with the establishment: avatar initials, name, establishment-assigned title (Provider Rank context), specialties, and rating. Each card does not yet link to the therapist's own Massage Nexus profile; that link is a known gap against the Spa Profile Guide's therapist-card requirements.

--------------------------------------------------------------------------------

7. Facilities
===============

Placement: Main column, after Therapists.

Currently shows Treatment Areas only (name, privacy level, capacity, and a short note per area). General facility/amenity display lives in the Overview section instead of here; consolidating both under one heading, or renaming this tab, is an open naming question.

--------------------------------------------------------------------------------

8. Reviews
============

Placement: Main column, after Facilities.

The section now reads approved Review records from review_main and their linked 1-to-10 rating events from rating_main. Rating count and review count remain separate. Until the Rating System's public XCES threshold and calculation are implemented, the profile says "Not enough ratings yet" instead of presenting an unexplained fixture average. Review cards show the public byline or anonymous label, publication date, exact score out of 10, title, and short description without star imagery. "Write a review" opens the authenticated Spa review form, and "View all reviews" opens the public list filtered to this Spa.

--------------------------------------------------------------------------------

9. Location and Contact
=======================

Placement: Main column, after Reviews.

A map tile opens a coordinate-based external map destination. The section shows the public address,
entrance directions, parking information, a Get Directions action, and nearby landmarks with
walking time. A Business Contact Channels panel presents repeatable public telephone, messaging,
website, and social-account actions. Telephone actions remain in the current page; external web
channels open separately with nofollow, noopener, and noreferrer protection. Unsupported stored
URL schemes are removed before rendering. A real embedded map is not implemented.

--------------------------------------------------------------------------------

10. Verification
==================

Placement: Main column, final section, not part of the tab bar.

A compact strip noting claimed/owner-managed status and the last-confirmed date. This does not yet cover source notes or a fuller verification history as described in the Spa Profile Guide's About, Verification, and Sources section.

--------------------------------------------------------------------------------

11. Booking Panel
===================

Placement: Right rail, sticky while scrolling on wide viewports; stacks below the main column on narrow viewports.

A quick-booking form: service selector (populated from the establishment's menu items with price shown inline), date picker, time selector, a Book Now submit button, a note that the user will be asked to log in or sign up, and a note that requests are confirmed by the spa rather than instantly guaranteed. This matches the Spa Profile Guide's Quick Booking direction and its Inquiry Only / Request Approval booking modes for a claimed-but-not-fully-verified establishment; Instant Confirmation, therapist selection, and group participants are not yet represented here.

--------------------------------------------------------------------------------

12. Dynamic Engagement Sidebar
==============================

Placement: Right rail, immediately below the Booking Panel.

A dynamically populated stack of engagement widgets (Trending Spas, Trending Therapists, Recent Reviews, Quizzes, Surveys, etc.) that scales according to the length of the main profile content. The widgets encourage cross-navigation and interaction. Anonymous users can participate in quizzes and surveys via session tracking.

--------------------------------------------------------------------------------

Spa Profile UI Behavior Notes
================================

The Spa identity, services, staff, facilities, location/contact, claim status, booking mode, and verification labels still render from synthetic sample data (see apps/web/app/Support/Demo/SampleContent.php) rather than establishment_main. The administration form and establishment structure guide now support public address, coordinates, directions, parking, landmarks, and repeatable public business contact channels, but the public-profile persistence connection remains a separate checklist item. The Review section reads approved review_main and rating_main records associated with the demo target's stable placeholder ID. Booking form submission does not yet call a booking endpoint. Empty Review state is implemented; broader section visibility rules remain deferred because the other sample sections are populated.

Open Items
============

- Reconcile the tab set with the Spa Profile Guide's eleven-section list once Gallery, Promotions and Updates, and a dedicated About/Verification/Sources tab are prioritized.
- Link therapist cards to their own profile pages once the Therapist Profile UI exists.
- Decide whether the coordinate-based external map action should remain the default or be supplemented by an embedded map after a provider is selected.
- Add a Report action once the moderation/reporting flow is designed.
- Confirm whether Facilities should absorb the Overview amenities list or remain Treatment-Areas-only.
