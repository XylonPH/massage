Title: Massage Nexus Site Structure Map
Document Version: 2.57
Date Created: 2026-07-10
Revision Date: 2026-07-21

Document Description:
This document maps the public website structure, profile page areas, content destinations, user-facing paths, unified workspace areas, system-facing routes, reserved route keywords, future Massage Campus areas, future Branded Portal entry points, and the accepted Booking and Appointment System surfaces for Massage Nexus. It is a site structure map, not a visible navigation menu, database schema, implementation schedule, or SEO XML sitemap.

Document Relationships:
This map is a route inventory across the project and therefore does not attempt to list every
related document. Each route family is governed by the corresponding capability, workspace,
interface, policy, or platform document in docs. Where this map and a governing capability
document differ on behavior, the governing capability document controls and this map must be
synchronized.

================================================================================
0. ROUTE CONVENTIONS
================================================================================

Primary domain:
massagenexus.com


Profile and section navigation behavior:
- every important profile section keeps a real server-rendered URL;
- opening a section URL directly returns a complete page for users and search engines;
- after the initial page load, Livewire navigation may fetch the next section in the background and replace the page content without a traditional full browser refresh;
- lazy loading may be used for heavy maps, galleries, reviews, and analytics;
- JavaScript enhancement must not make the direct URL unusable.


Default public site:

2A. ACCOUNT, REGISTRATION, LOGIN, AND VERIFICATION
================================================================================

Public account routes:
/register
/login
/logout
/forgot-password
/reset-password
/verify-email
/resend-verification
/complete-profile

Rules:
- ordinary public browsing does not require registration;
- Massage Nexus membership is 18+;
- registration creates a shared network account and Massage Nexus membership;
- an unverified user cannot enter the normal User Workspace;
- username or email may be used to log in;
- password managers and browser autofill must work;
- policy acceptance and privacy acknowledgment are versioned;
- sensitive recovery responses must not reveal account existence unnecessarily.

No public /account route is required.
Private account management remains under /workspace.

47. BOOKING ROUTE BOUNDARIES
================================================================================

- Public spa and therapist booking pages explain availability, booking enablement, policies,
  unclaimed-profile limits, and supported actions.
- Personal Appointment Plans, private intake, booking history, payment, refund, loyalty, and
  personal analytics remain under /workspace.
- Provider scheduling, assignment, queue, walk-in, intake templates, eligibility, finance, and
  compensation remain permission-aware workspace areas.
- No public URL exposes a client's private intake or another provider's booking history.
- Unclaimed providers do not receive confirmed Massage Nexus reservations or platform payment.
- API routes use the same server-side authorization, eligibility, policy, privacy, and retention
  rules as the website.
- Booking is service-flexible but initially massage-, spa-, bodywork-, beauty-, and
  wellness-aware.
