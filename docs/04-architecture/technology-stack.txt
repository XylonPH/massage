DOCUMENT DESCRIPTION
--------------------
This document records the accepted and working technology direction for Massage Nexus. Exact package versions are locked through each application's dependency and lock files when implementation begins. Versions shown in planning notes must not be treated as installed facts until the local environment and compatibility requirements are verified. This file defines the intended stack and major boundaries; the File and Folder Structure document defines where the code and supporting files belong.

7. STORAGE AND MEDIA SECURITY
================================================================================

Development Public Media:

Development Private Documents:
apps/web/storage/app/private/

Development Restricted Media:
apps/web/storage/app/restricted/

Upload Quarantine:
apps/web/storage/app/quarantine/

Temporary Uploads:
apps/web/storage/app/temporary/

Production Storage Containers or Logical Areas:
- public-media;
- private-document;
- restricted-media;
- quarantine;
- temporary-upload;
- backup.

Private-file rules:
- never place private files under apps/web/public/;
- never expose the original private storage path in HTML or JavaScript;
- check MediaPolicy or the relevant policy before delivery;
- stream the file or issue a short-lived protected URL;
- use UUID or ULID identifiers rather than predictable numeric file paths;
- disable directory listing;
- prevent public CDN caching of restricted files;
- record sensitive access where auditing is justified.

Native screenshot blocking may be used as an additional protection on supported mobile or desktop platforms. It is not the primary access-control system and cannot prevent a separate camera from photographing a screen.
