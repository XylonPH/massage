================================================================================
1. PURPOSE
================================================================================

The desktop application is an installed workspace for:
- administrators;
- researchers;
- translators;
- moderators;
- support agents;
- editors;
- provider operators;
- data managers;
- authorized enterprise users.

It emphasizes:
- multi-window work;
- keyboard efficiency;
- bulk operations;
- large datasets;
- local files;
- offline work;
- long-running tasks;
- advanced media and document handling.

It uses the same server-side permissions and business rules as web and mobile.

================================================================================

- multi-window workspace;

- local document comparison;
- high-volume media organization;
- batch metadata editing;

and local file systems.

7. BULK IMPORT AND EXPORT
================================================================================

Possible formats:
- CSV;
- JSON;
- spreadsheet;
- media package;
- translation package;
- research package;
- report package.

Import:
1. select source;
2. validate schema;
3. preview;
4. identify duplicates and conflicts;
5. map fields;
6. apply permissions;
7. dry run;
8. submit;
9. review result;
10. preserve audit.

Bulk import never bypasses validation or silently overwrites verified data.

================================================================================
8. OFFLINE WORKSPACE
================================================================================

Offline-capable:
- assigned research;
- translation packages;
- draft content;
- selected Support drafts;
- approved provider operations;
- local exports;
- public reference data.

States:
- local;
- pending;
- syncing;
- conflict;
- failed;
- completed.

Sensitive offline data:
- is encrypted;
- may expire;
- supports remote revocation;
- excludes unnecessary records;
- clears after account removal or policy.

The desktop application does not maintain an uncontrolled full production database replica.

================================================================================
9. LOCAL FILE SYSTEM
================================================================================

Possible areas:
- user-selected workspace;
- import;
- export;
- temporary processing;
- cache;
- logs;
- encrypted local database.

Rules:
- show storage location;
- avoid hidden permanent copies;
- support cleanup;
- protect sensitive files;
- keep secrets out of ordinary text;
- validate paths;
- prevent traversal;
- handle removable drives carefully.

================================================================================
10. PLATFORM AND UPDATE MODEL
================================================================================

Windows is the likely initial platform.

The capability specification does not bind the application to a particular desktop framework.

Later desktop platforms use the same API and permissions.

Update system:
- signed packages;
- integrity verification;
- version check;
- release notes;
- staged rollout;
- rollback or reinstall path;
- minimum supported version;
- forced update only for security or compatibility.

Do not execute unsigned production code.

================================================================================
11. SECURITY
================================================================================

Use:
- passkeys or secure authentication;
- encrypted token storage;
- device registration;
- session revocation;
- signed application;
- encrypted sensitive local data;
- automatic lock;
- safe logs;
- least privilege;
- server permission recheck;
- secure updates;
- export controls.

A local installation is not automatically trusted.

13. PLATFORM BOUNDARIES
================================================================================

Web is better for:
- canonical public pages;
- search indexing;
- public Help and policy;
- immediate browser access;
- public sharing.

Mobile is better for:
- location;
- camera field work;
- push;
- QR;

Desktop focuses on high-volume, multi-window, local-file, bulk, and offline work.

15. CURRENT DIRECTIONS
================================================================================

- Desktop is an installed high-volume operational and administrative client.
- It uses the same server API, permissions, and business rules as web and mobile.
- Its strengths are multi-window work, keyboard operation, bulk import and export, local files,
  media processing, large data review, and encrypted offline workspaces.
- It does not replace the canonical public web application.
- It does not maintain an uncontrolled production database copy.
- Windows is the likely initial platform without embedding a framework decision here.
